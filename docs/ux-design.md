# Coffee Brewing Notebook

## Product Design Manifesto

Version 1.0

---

# Introduction

This document defines the design philosophy of the application.

It is **not** a specification of screens or components. Instead, it explains the identity of the product, the experience it should create, and the principles that should guide every design decision.

This application is not a timer, a recipe database, or a coffee journal.

It is a notebook for coffee experiments.

Every interaction should reinforce that identity.

---

# Product Vision

The application helps advanced home baristas improve their coffee through deliberate experimentation.

Users are curious by nature. They enjoy adjusting one parameter at a time, observing the result, and slowly refining a recipe until it matches their personal preferences.

The application should support this process without becoming part of the work.

It should feel like a quiet companion that remembers every experiment and helps users understand what they have learned.

The application is not trying to replace the user's intuition.

It should strengthen it.

---

# Design Principles

## The coffee comes first

The application should never compete with the brewing ritual.

When someone is brewing coffee, their attention belongs to the kettle, the brewer, the scale, and the coffee.

The application should remain almost invisible.

It provides the right information at the right moment and then gets out of the way.

---

## Minimize cognitive load

Brewing already requires attention.

The interface should never ask users to process unnecessary information.

Every screen should answer one question.

Examples:

* What am I brewing?
* What should I do next?
* How was the result?
* What changed since last time?

Avoid combining multiple goals on a single screen.

---

## Encourage experimentation

The application should reward curiosity.

Instead of storing recipes, it should tell the story of how recipes evolve.

Users should naturally ask:

> What happens if I grind one click finer?

The application should make that experiment easy to perform and easy to understand afterwards.

---

## Progress over perfection

Coffee recipes are never truly finished.

The interface should celebrate improvement rather than perfection.

Users should feel they are building knowledge over time.

Every brew becomes another page in a long notebook.

---

## Calm interface

The interface should create mental space.

There should be generous spacing.

Clear hierarchy.

Very few colors.

Minimal decoration.

Animations should be subtle.

Nothing should feel rushed.

---

# Emotional Goals

The application should create three emotions.

## Focus

During brewing, only relevant information is visible.

Everything unrelated disappears.

The interface helps users stay present.

---

## Relaxation

The application should never create pressure.

No achievements.

No streaks.

No gamification.

No artificial urgency.

Brewing coffee is already enjoyable.

The application should preserve that feeling.

---

## Progress

Users should constantly feel that they are becoming better brewers.

The application should transform past experiments into future recommendations.

Knowledge accumulates naturally.

---

# Personality

If the application were a person, it would be:

* calm
* knowledgeable
* discreet
* precise
* curious

It would never be loud.

It would never try to impress.

It quietly helps users improve.

---

# Visual Identity

The interface should resemble premium tools rather than coffee shops.

Inspirations include:

* Timemore
* Leica
* Apple
* Muji
* Field Notes
* high-quality printed brewing guides

The aesthetic is modern, restrained, and timeless.

Avoid visual clichés associated with coffee.

Avoid excessive use of brown colors, coffee bean illustrations, burlap textures, or café-inspired graphics.

Coffee already provides warmth.

The interface should provide clarity.

---

# Color Philosophy

Use a restrained palette.

Backgrounds should feel soft rather than bright.

Cards should create subtle separation.

Accent colors should be used only to guide attention.

Color communicates importance, not decoration.

Success and failure should rely on hierarchy and wording before relying on color.

---

# Typography

Typography is one of the strongest visual elements.

Recipe names should feel important.

Numbers should be highly legible.

Brewing parameters deserve visual prominence.

Supporting information should remain quiet.

The overall feeling should resemble reading a well-designed notebook.

---

# Layout

Whitespace is an active design element.

Components should never compete for attention.

Large margins improve focus.

The interface should breathe.

Information density should remain low.

Whenever possible, reduce rather than add.

---

# Interaction Design

Every interaction should require intention.

Avoid accidental complexity.

The application should favor:

* taps
* simple selections
* progressive disclosure

Avoid long forms whenever possible.

Advanced options should appear only when requested.

---

# Progressive Disclosure

Most brewing sessions should require very little interaction.

Only after completing a brew should users have the opportunity to explore additional details.

Complexity should always remain optional.

The application should adapt to curiosity rather than requiring it.

---

# Recording a Brew

Recording should feel effortless.

The user should never feel like they are filling in a database.

Instead, it should feel similar to writing a short note inside a notebook.

The goal is memory, not administration.

---

# Evaluating Coffee

Evaluation exists to improve future brews.

It is not an examination.

The application should first ask whether the brew was acceptable.

Only afterwards should it ask how much it was enjoyed.

Detailed sensory analysis should remain optional.

This reflects how experienced home baristas naturally evaluate coffee.

---

# History

History is not an archive.

History is evidence.

Every previous brew should help answer future questions.

The interface should reveal trends, improvements, and recurring patterns.

Users should quickly understand:

* what changed
* why it changed
* whether it improved

---

# Comparisons

Comparison is one of the application's greatest strengths.

Rather than presenting two recipes independently, comparisons should emphasize differences.

Examples include:

* grind size changed
* water temperature changed
* extraction time changed
* enjoyment increased
* bitterness decreased

The application should help users understand relationships rather than memorize values.

---

# Motion

Animations should reinforce the brewing ritual.

Motion should communicate:

* transition
* completion
* progress

Never excitement.

Animations should feel smooth and intentional.

If an animation attracts attention to itself, it is probably too strong.

---

# Writing Style

The interface should speak like a knowledgeable brewing companion.

It should be concise.

Helpful.

Never overly enthusiastic.

Avoid marketing language.

Avoid technical jargon unless it genuinely helps understanding.

Prefer:

"Try grinding one click finer."

Instead of:

"Extraction efficiency appears suboptimal."

The tone should remain calm and confident.

---

# What This Application Must Never Become

The application must never feel like:

* laboratory software
* spreadsheet software
* enterprise data entry
* social media
* productivity software
* a game

Users should never feel they are completing administrative work.

Every interaction should reinforce the pleasure of brewing coffee.

---

# Success Criteria

A successful design allows users to:

* begin brewing within seconds
* record experiments effortlessly
* understand what changed between brews
* identify progress over time
* enjoy using the application every morning

The application should quietly become part of the brewing ritual.

When users think about improving their coffee, opening the application should feel as natural as reaching for their scale or grinder.

That is the experience this product should create.
